## Project Overview

PUSE is a save editor for Pokemon Unbound (v2.1.1.1). It has two runtime modes:
- **Local mode** (`VITE_RUNTIME_MODE=local`): all parsing/editing runs in-browser (JS)
- **Backend mode** (`VITE_RUNTIME_MODE=backend`): uses FastAPI at `http://localhost:8000`

A Nintendo Switch homebrew port is in progress under `switch-homebrew/`.

## Commands

### Frontend (run from `frontend/`)

```bash
npm install
npm run dev          # dev server at http://localhost:5173
npm run build        # production build
npm run lint         # lint all files
npm run parity:all   # run all regression scripts (rtc, pc-fallback, identity)
npm run icons:manifest  # regenerate icon manifests after changing pinned commits
```

### Backend (run from `backend/`)

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### Docker (full stack)

```bash
docker compose up --build
```

### Switch homebrew (from `switch-homebrew/`)

```bash
switch-homebrew/scripts/build_docker.sh   # full Docker build + SD bundle
switch-homebrew/scripts/sync_romfs_data.sh  # sync backend/data -> romfs/data
```

There is no configured test runner (`package.json` has no test script). Regression scripts live in `frontend/scripts/` and are run via the `npm run parity:all` / individual npm scripts above.

## Architecture

### Dual runtime parity

`frontend/src/services/apiClient.js` is the key abstraction. It exports `createApiClient(runtimeMode)` which returns either:
- A **backend client** that calls FastAPI REST endpoints
- A **local client** that imports and calls JS modules in `frontend/src/core/`

The `frontend/src/core/` directory is a JS port of the backend Python logic with behavioral parity as the goal. Every core module has a backend counterpart:

| Frontend | Backend |
|---|---|
| `core/party.js` | `modules/party.py` |
| `core/pc.js` | `modules/pc.py` |
| `core/bag.js` | `modules/bag.py` |
| `core/money.js` | `modules/money.py` |
| `core/sections.js` | `core/sections.py` |
| `core/checksum.js` | (inline in backend modules) |
| `core/rtc.js` | `tools/rtc_*.py` |
| `core/saveConvert.js` | `tools/save_convert.py` |

**When behavior differs, backend is authoritative.** See `switch-homebrew/RULES.md` section 2.

### Backend save lifecycle

`backend/main.py` holds all save state in `current_save["data"]` (in-memory bytearray). Edits accumulate without writing to disk. `/save-all` commits: recalculates checksums for trainer section (id=1), bag sectors (13-16), PC buffer sectors, and absolute-touched sectors, then writes `edited_save.sav`. `/download` serves the file.

### Frontend state flow

`App.jsx` owns global state: loaded save, active tab, selected Pokemon, PC box. All lazy-loaded components (`PartyGrid`, `PCGrid`, `BagView`, `PokemonEditorModal`, `AddPcPokemonModal`) receive `client` (the apiClient instance) as a prop. Save/download calls go through `client.saveAll()` then `client.downloadSave()`.

### Static data

ROM-extracted lookup tables live in `backend/data/` and are mirrored into `frontend/src/core/` (JSON imports) and `switch-homebrew/romfs/data/` (via sync script). Never edit data files by hand — they are generated by tools in `backend/tools/`.

### Switch homebrew

`switch-homebrew/` contains a Plutonium (libnx) C++ port, currently in development. Build requires Docker with devkitPro. The backend Python is the canonical behavior reference. See `switch-homebrew/RULES.md` and `switch-homebrew/PLAN.md` for full requirements.

## Key Conventions

### API contracts are stable
Payload keys (`nature_id`, `item_id`, `ability_index`, `current_ability_index`, etc.) are shared contracts between frontend and backend. Change both sides together or don't change at all.

### Checksum after every mutation
Any code path that writes save data must end with checksum recalculation. In backend: `bag_mod.gba_checksum` / `bag_mod.recalculate_checksum` / `party_mod.wu16`. In frontend local mode: `checksum.js`.

### Legit mode
`App.jsx` has a `legitMode` flag (persisted to localStorage). Components receive it as a prop. It enforces 510 EV cap and restricts some edits. Don't bypass it in new features.

### Personal save files
Keep `.sav`/`.srm`/ROM files under `backend/local_artifacts/` (gitignored). Never commit save files.

### Icon assets are optional
Neither backend nor frontend requires icons to run. Missing icons produce a placeholder. Icon manifests at `frontend/src/data/pokemon-icon-manifest.json` and `frontend/src/data/item-icon-manifest.json` are committed.
